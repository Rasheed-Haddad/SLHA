# وثيقة مصالحة رسمية 🤍

صفحة وثيقة مصالحة بستايل عتيق رسمي، مع توحيد البيانات بين الطرفين عبر MongoDB،
وتصدير PDF. صديقتك تفتح **رابطاً فقط** — لا تتعامل مع أي شيء تقني.

---

## 1) تشغيل محلي (للتجربة على جهازك)

1. ثبّت [Node.js](https://nodejs.org) (نسخة 18 أو أحدث).
2. في هذا المجلد، انسخ ملف `.env.example` وأعد تسميته إلى `.env`.
3. افتح `.env` وضع فيه سلسلة الاتصال من Atlas:
   - من Atlas: زر **Connect → Drivers → Node.js**، انسخ السلسلة.
   - استبدل `<password>` بكلمة سر مستخدم القاعدة.
4. شغّل:
   ```
   npm install
   npm start
   ```
5. افتح المتصفح على: http://localhost:3000

> ملاحظة: لن تصل صديقتك إلى localhost. للوصول من جهازها، انشر المشروع (الخطوة 2).

---

## 2) النشر المجاني على Render

1. ارفع هذا المجلد إلى مستودع على GitHub (بدون ملف `.env` — هو مُستثنى تلقائياً).
2. ادخل [render.com](https://render.com) → **New → Web Service** → اربط المستودع.
3. الإعدادات:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
4. في تبويب **Environment**، أضف المتغيرات:
   - `MONGODB_URI` = سلسلة الاتصال من Atlas
   - `DB_NAME` = `reconciliation` (اختياري)
   - `COLLECTION` = `documents` (اختياري)
5. اضغط **Deploy**. بعد دقائق يعطيك رابطاً مثل:
   `https://your-app.onrender.com`

أرسلي هذا الرابط لصديقتك. كلاكما يفتح نفس الرابط ويعبّي نصيبه ويوقّع. ✨

---

## 3) إعداد Atlas (مرة واحدة)

1. في Atlas → **Database Access**: أنشئ مستخدماً بكلمة سر (ستضعها في `MONGODB_URI`).
2. في Atlas → **Network Access**: أضف `0.0.0.0/0` (السماح من أي مكان) حتى يصل Render.
3. القاعدة والمجموعة تُنشآن تلقائياً عند أول حفظ — لا حاجة لإنشائهما يدوياً.

---

## كيف يعمل القفل؟

- كل طرف يكتب اسمه، يرفع صورة توقيعه، ويضغط زر التثبيت → تُقفل بياناته.
- عند توقيع **الطرفين** → تُقفل الوثيقة **نهائياً** ويظهر الختم الرسمي.
- السيرفر يمنع أي تعديل بعد التثبيت (حماية من الكتابة فوق الطرف الآخر).
